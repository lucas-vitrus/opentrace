# Security Policy

## Supported Versions

Trace is currently in active development. Security updates are provided for the latest release only.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |

We recommend always using the latest version of Trace. Download it at [buildwithtrace.com/download](https://buildwithtrace.com/download).

## Reporting a Vulnerability

If you discover a security vulnerability in Trace, please report it responsibly. **Do not open a public GitHub issue.**

### How to Report

Email us at: **hello@buildwithtrace.com**

Include the following in your report:
- A description of the vulnerability
- Steps to reproduce the issue
- Affected versions (if known)
- Any potential impact or severity assessment

### What to Expect

- **Acknowledgment:** We will acknowledge receipt of your report within 48 hours.
- **Updates:** We will provide status updates as we investigate, typically within 7 days.
- **Resolution:** If the vulnerability is confirmed, we will work on a fix and coordinate disclosure with you.
- **Credit:** We are happy to credit researchers who report valid vulnerabilities (unless you prefer to remain anonymous).

### Scope

This policy applies to the Trace application and its official repositories. Third-party plugins, forks, or community extensions are not covered.

## Responsible Disclosure

We ask that you:
- Give us reasonable time to address the issue before public disclosure
- Avoid accessing or modifying user data without permission
- Act in good faith to avoid disruption to Trace users

Thank you for helping keep Trace secure.
